# Dear Diary - Integrated Project

Dear Diary is a mobile-first psychological horror game website featuring an interactive diary and community hub where users can click on mysterious items to uncover clues about the missing character Wei Yi and discuss theories with other players. The website blends interactive storytelling with community engagement for an immersive experience.

## Design Process

The design focuses on creating a haunting, personal atmosphere similar to discovering someone's private diary, while incorporating interactive elements inspired by modern gaming websites. The website is designed primarily for mobile devices with a responsive container that mimics a phone interface. The target audience includes horror game enthusiasts who enjoy unraveling mysteries through interactive storytelling and community discussion on their mobile devices.

### User Stories
* As a mystery solver, I want to click on diary items marked with "?", so that I can uncover hidden clues like in interactive game websites
* As a lore enthusiast, I want to explore character files through interactive reveals, so that I can understand their backstories progressively
* As a community member, I want to post and read theories in the Community Hub, so that I can discuss the game with others
* As a mobile user, I want the website to be optimized for my phone screen with touch-friendly interactions, so that I can navigate easily
* As a new visitor, I want to sign up for the newsletter, so that I can receive exclusive updates

### Wireframes
* Based on provided Figma designs showing the mobile-optimized diary interface and Community Hub
* Mobile-first design approach with 450px max-width container
* Interactive elements inspired by gaming website interfaces

## Features

### Existing Features
* **Mobile-First Structure** - HTML files optimized for mobile with 450px container
* **Mobile Navigation** - Sandwich menu and login button designed for touch screens
* **Horror-themed Design** - Dark color scheme with blood-red accents and paper textures
* **Responsive Container** - Centered mobile layout with side borders for phone-like experience
* **Community Hub Structure** - Basic forum layout with discussion cards ready for content

### Features Left to Implement
* **Interactive Diary** - Clickable "?" items that reveal clues through modal popups, inspired by investigative interfaces
* **Character Profiles** - Expandable character files with detailed backstories revealed through interaction
* **Community Forum** - Discussion cards for fan theories with interactive elements
* **Authentication** - Login/Signup forms with validation
* **Merchandise Store** - Product listings and shopping functionality
* **JavaScript Interactivity** - All interactive functionality for mobile touch, inspired by game lore exploration websites

## Testing

### Testing To Be Conducted
1. **Mobile Responsiveness**:
   - Test on various phone screen sizes (iPhone, Android)
   - Verify touch targets are large enough (minimum 44x44px)
   - Check layout on different mobile browsers

2. **Interactive Diary**:
   - Test "?" item clicks on touch devices
   - Verify modal popups work with touch interactions
   - Test interactive clues reveal like investigative game interfaces

3. **Community Hub**:
   - Test discussion card layouts on mobile
   - Verify posting functionality works on mobile
   - Check image display in forum posts

4. **Form Validation**:
   - Test login form on mobile keyboards
   - Test form submission on mobile data connections
   - Verify input fields are mobile-friendly

5. **Navigation Testing**:
   - Test sandwich menu on touch screens
   - Verify all links work on mobile
   - Check page load times on mobile networks
